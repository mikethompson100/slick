Example 15 - customPaging

Demonstrating the use of customPaging to perform special operations on the dots.
The value of i in the function iterates through and builds the image path and filenames of the numerical images.

 $('.carousel15').slick({
   dots: true,
   customPaging : function(slider, i) {
       return $('<img src="img/0' + (i+1) 
       + 'dots.svg">').text(i + 1);
   }
 });
  
  Example:   <img src="img/02dots.svg">

Example 17 - edgeFriction

When swiping or dragging beyond a non-infinite carousel's edge, edgeFriction allows for a specific visual kickback amount.
An integer is required. The default value is .15


    $('.carousel17').slick({
      edgeFriction: 0.5,
      infinite: false  (required for edgeFriction)
    });

Example 18 - lazyLoad

Loading carousel images into the browser on a 'needed' basis and not all at page load time for efficiency.
Speed: 100 is chosen to exaggerate the loading effect (will flash white). Once all 6 are loaded it returns to the usual smooth transition.


    $('.carousel18').slick({
      lazyLoad: 'ondemand',
      speed: 100
    });

Example 19 - lazyLoaded/lazyLoadError events

Events for managing lazyloading. Message displays announcing the success or failure of each image load.
Note: 'infinite: false' has also been added, otherwise the carousel seems to load the last image as well. Not sure if it is a bug or feature.



  $('#carousel19').slick({
    lazyLoad:'ondemand',
    speed: 100,
    infinite: false
  });
  
  $('#carousel19').on('lazyLoaded', function (e, slider, image, imageSource) {
      var messageLocation = document.getElementById("demo");
      const node = document.createElement("div");
      node.innerHTML = imageSource + " loaded on the page.";
      messageLocation.appendChild(node);
  });
  
  $('#carousel19').on('lazyLoadError', function (e, slider, image, imageSource) {
    var messageLocation = document.getElementById("demo");
    const node = document.createElement("div");
    node.innerHTML = imageSource + " failed to load successfully.";
    messageLocation.appendChild(node);
  });

Example 20 - mobileFirst/initialSlide

The default for mobileFirst is false which activates the responsive settings for mobile. When mobileFirst is switched to true, responsive settings are for desktop.
In this example, initialSlide is showcased as the differentiating feature between mobile and desktop that selects the starting slide position.

    $('.carousel20').slick({
      mobileFirst: true,
      initialSlide: 5,
      responsive: [
        {
          breakpoint: 768,
          settings: {
            initialSlide: 0
          }
        }
       ]
    });

Example 21 - pauseOnFocus / pauseOnHover / pauseOnDotsHover

Examples of hovering features which require autoplay: true. The two defaults show autoplay pausing on hovering and also focusing either on the slide image itself or the arrows that control the slides.
The pauseOnDotsHover requires dots: true and pauses the slideshow when the dots are hovered or focused on. On mobile view, only focusing on the dots causes the pause.


  $('#carousel21').slick({
    autoplay: true,
    pauseOnFocus: true,  (default)
    pauseOnHover: true,  (default)
    dots: true,
    pauseOnDotsHover: true
  });
